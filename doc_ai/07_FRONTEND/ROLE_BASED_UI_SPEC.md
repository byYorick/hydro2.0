## Обзор ролей

### Роли пользователей

1. **Агроном** (`agronomist`)
   - Фокус: рецепты, фазы роста, анализ эффективности
   - Основные задачи: мониторинг культур, настройка рецептов, анализ урожайности

2. **Админ** (`admin`)
   - Фокус: полное управление системой
   - Основные задачи: управление пользователями, настройки системы, общий контроль

3. **Инженер** (`engineer`)
   - Фокус: техническое обслуживание, диагностика
   - Основные задачи: диагностика устройств, настройка узлов, решение технических проблем

4. **Оператор** (`operator`)
   - Фокус: ежедневное управление зонами, оперативные действия
   - Основные задачи: управление зонами, ручной полив, дозирование, разрешение тревог
   - Ограничения: не может менять пользователей и системные настройки

5. **Наблюдатель** (`viewer`)
   - Фокус: только просмотр данных
   - Основные задачи: мониторинг состояния системы, просмотр данных, история
   - Ограничения: не может создавать команды и менять настройки

## Dashboard для каждой роли

### Dashboard агронома

**Приоритет:** Мониторинг культур и рецептов

#### Основные виджеты:

1. **Обзор зон по культурам**
   ```
   ┌─────────────────────────────────────┐
   │  Салат (3 зоны)                    │
   │  [████████████░░] 75% готовности   │
   │  Фаза: Вегетация (день 12/21)      │
   └─────────────────────────────────────┘
   ```
   - Группировка зон по культурам
   - Прогресс фаз для каждой культуры
   - Средние метрики (pH, EC) по культуре

2. **Ключевые метрики всех зон**
   - Агрегированные pH/EC по всем активным зонам
   - Тренды за последние 24 часа
   - Индикаторы отклонений от нормы

3. **Активные рецепты**
   - Список применяемых рецептов
   - Прогресс по каждой фазе
   - Следующие переходы фаз (таймеры)

4. **Аналитика эффективности**
   - Сравнение рецептов
   - Прогноз урожайности
   - Рекомендации по оптимизации

5. **Последние события по культурам**
   - Фильтрация по типам культур
   - Важные изменения в зонах

#### Быстрые действия:
- Создать новый рецепт
- Применить рецепт к зоне
- Сравнить рецепты
- Экспорт аналитики

### Composable для ролей

#### useRole.ts
```typescript
import { computed } from 'vue'
import { usePage } from '@inertiajs/vue3'

export function useRole() {
  const page = usePage()
  const user = computed(() => page.props.auth?.user)
  
  const role = computed(() => user.value?.role)
  const isAgronomist = computed(() => role.value === 'agronomist')
  const isAdmin = computed(() => role.value === 'admin')
  const isEngineer = computed(() => role.value === 'engineer')
  const isOperator = computed(() => role.value === 'operator')
  const isViewer = computed(() => role.value === 'viewer')
  
  const canEdit = computed(() => 
    isAdmin.value || isAgronomist.value || isEngineer.value || isOperator.value
  )
  const canManageUsers = computed(() => isAdmin.value)
  const canManageSystem = computed(() => isAdmin.value)
  const canDiagnose = computed(() => isEngineer.value || isAdmin.value)
  const canCreateCommands = computed(() => 
    isAdmin.value || isAgronomist.value || isEngineer.value || isOperator.value
  )
  const canEditRecipes = computed(() => 
    isAdmin.value || isAgronomist.value || isOperator.value
  )
  const canResolveAlerts = computed(() => 
    isAdmin.value || isAgronomist.value || isEngineer.value || isOperator.value
  )
  const canViewOnly = computed(() => isViewer.value)
  
  return {
    role,
    isAgronomist,
    isAdmin,
```

## Стилизация ролевых интерфейсов

### Цветовые акценты (опционально)

- **Агроном:** Зеленые акценты (природа, рост)
- **Админ:** Синие акценты (управление, контроль)
- **Инженер:** Оранжевые/желтые акценты (техника, диагностика)
- **Оператор:** Голубые акценты (оперативность, действия)
- **Наблюдатель:** Серые акценты (нейтральность, просмотр)

### Иконки

- **Агроном:** 🌱 растения, 📊 аналитика
- **Админ:** 👥 пользователи, ⚙️ настройки
- **Инженер:** 🔧 инструменты, 📊 метрики
- **Оператор:** ⚡ быстрые действия, 🎯 управление
- **Наблюдатель:** 👁️ просмотр, 📈 мониторинг
