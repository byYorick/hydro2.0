# Руководство пользователя: Просмотр мониторинга

Это руководство поможет вам получить доступ к системе мониторинга и просматривать метрики сервисов.


Compatible-With: Protocol 2.0, Backend >=3.0, Python >=3.0, Database >=3.0, Frontend >=3.0.
Breaking-change: обратная совместимость со старыми форматами и алиасами не поддерживается.

---

## 🚀 Быстрый старт

### 1. Доступ к Grafana

**URL:** `http://localhost:3000` (или IP вашего сервера)

**Учетные данные:**

#### Development окружение:
- **Логин:** `admin`
- **Пароль:** `admin` (при первом входе потребуется сменить)

#### Production окружение:
- **Логин:** `admin` (или значение из `GRAFANA_ADMIN_USER`)
- **Пароль:** значение из переменной окружения `GRAFANA_ADMIN_PASSWORD`

> ⚠️ **Важно:** В production обязательно установите безопасный пароль через переменную окружения `GRAFANA_ADMIN_PASSWORD`.

## 📊 Доступные Dashboards

### Автоматический импорт

Dashboards должны автоматически импортироваться при запуске Grafana (если настроен provisioning). После входа в Grafana вы увидите следующие готовые дашборды:

### Ручной импорт (если dashboards не видны)

Если dashboards не появились автоматически:

1. В меню слева выберите **Dashboards** → **Import**
2. Нажмите **Upload JSON file**
3. Выберите файл из `backend/configs/{env}/grafana/dashboards/`:
   - `history-logger.json` - для History Logger Service
   - `automation-engine.json` - для Automation Engine Service
4. Нажмите **Load** → **Import**

**Примечание:** После первого запуска Grafana может потребоваться несколько секунд для автоматического импорта dashboards. Если dashboards не появились, перезапустите контейнер Grafana.

---

### Список доступных Dashboards:

### 1. **History Logger Service**
   - **Путь:** Dashboards → History Logger Service
   - **Описание:** Мониторинг сервиса записи телеметрии
   - **Что показывает:**
     - Статус сервиса и размер очереди
     - Поток телеметрии (получено/обработано)
     - Производительность обработки
     - Ошибки и потерянные сообщения
     - Регистрация узлов
     - Heartbeat сообщения

### 2. **Automation Engine Service**
   - **Путь:** Dashboards → Automation Engine Service
   - **Описание:** Мониторинг системы автоматизации зон
   - **Что показывает:**
     - Статус сервиса и скорость обработки
     - Время обработки зон
     - Команды по зонам и типам
     - Ошибки и проблемы
     - Эффективность работы

### 3. **System Overview**
   - **Путь:** Dashboards → System Overview
   - **Описание:** Общий обзор системы

### 4. **Alerts Dashboard**
   - **Путь:** Dashboards → Alerts Dashboard
   - **Описание:** Активные алерты системы

### 5. **Zone Telemetry**
   - **Путь:** Dashboards → Zone Telemetry
   - **Описание:** Телеметрия по зонам

### 6. **Node Status**
   - **Путь:** Dashboards → Node Status
   - **Описание:** Статус узлов

## 🔍 Как использовать Dashboards

### Навигация

1. **Открыть Dashboard:**
   - Нажмите на иконку меню (☰) слева
   - Выберите **Dashboards**
   - Найдите нужный dashboard в списке

2. **Обновление данных:**
   - Dashboards обновляются автоматически каждые 15 секунд
   - Можно обновить вручную кнопкой **↻** в правом верхнем углу
   - Можно изменить интервал обновления в выпадающем меню

3. **Изменение временного диапазона:**
   - В правом верхнем углу выберите период (Last 5 minutes, Last 1 hour и т.д.)
   - Или используйте календарь для выбора конкретного периода

### Понимание метрик

#### History Logger Dashboard

**Ключевые индикаторы:**

- **Service Status** (Статус сервиса)
  - 🟢 UP - сервис работает
  - 🔴 DOWN - сервис недоступен

- **Queue Size** (Размер очереди)
  - 🟢 < 5000 - нормально
  - 🟡 5000-10000 - внимание
  - 🔴 > 10000 - критично (переполнение)

- **Queue Age** (Возраст элементов)
  - 🟢 < 60 сек - нормально
  - 🟡 60-300 сек - задержка
  - 🔴 > 300 сек - критично

- **Processing Rate** (Скорость обработки)
  - Показывает количество сообщений в секунду
  - Должна быть близка к скорости получения

#### Automation Engine Dashboard

**Ключевые индикаторы:**

- **Service Status** (Статус сервиса)
  - 🟢 UP - сервис работает
  - 🔴 DOWN - сервис недоступен

- **Zone Checks Rate** (Скорость проверок зон)
  - Показывает, как часто проверяются зоны
  - Норма: несколько проверок в минуту

- **Commands Sent Rate** (Скорость отправки команд)
  - Показывает активность автоматизации
  - Высокое значение = много корректировок

- **Zone Check Duration** (Время обработки зон)
  - 🟢 P99 < 5 сек - отлично
  - 🟡 P99 5-30 сек - нормально
  - 🔴 P99 > 30 сек - медленно

## 📈 Работа с графиками

### Просмотр деталей

1. **Наведите курсор** на график для просмотра значений в конкретный момент времени
2. **Кликните на легенду** для скрытия/показа конкретной метрики
3. **Используйте зум** - выделите область на графике для детального просмотра

### Экспорт данных

1. Нажмите на название панели
2. Выберите **Panel options** → **Share**
3. Выберите формат экспорта:
   - **PNG** - изображение
   - **CSV** - данные для Excel
   - **JSON** - для программной обработки

## 🚨 Просмотр алертов

### В Grafana

1. Откройте **Alerts Dashboard**
2. Просмотрите список активных алертов
3. Кликните на алерт для деталей

### В Prometheus

**URL:** `http://localhost:9090/alerts`

1. Откройте Prometheus UI
2. Перейдите в раздел **Alerts**
3. Просмотрите все алерты с их статусами:
   - 🟢 **Inactive** - нет проблем
   - 🟡 **Pending** - проблема обнаружена, ожидает подтверждения
   - 🔴 **Firing** - активный алерт, требуется внимание

### Типы алертов

#### History Logger:
- **HistoryLoggerQueueOverflow** - очередь переполнена
- **HistoryLoggerDroppingMessages** - потеря сообщений
- **HistoryLoggerDatabaseErrors** - ошибки БД
- **HistoryLoggerNoProcessing** - сервис не обрабатывает данные
- **HistoryLoggerUnknownNodeEventsSpike** - рост неизвестных `node event` (проверьте `event_code` в прошивке/контракте)

#### Automation Engine:
- **AutomationEngineLoopErrors** - ошибки в цикле
- **AutomationEngineConfigFetchErrors** - проблемы с конфигурацией
- **AutomationEngineMQTTPublishErrors** - ошибки отправки команд
- **AutomationEngineNoZoneChecks** - не обрабатываются зоны

## 🔧 Прямой доступ к метрикам

### Prometheus UI

**URL:** `http://localhost:9090`

**Основные разделы:**

1. **Graph** - запросы метрик и построение графиков
2. **Alerts** - просмотр алертов
3. **Status** → **Targets** - статус источников метрик
4. **Status** → **Rules** - правила алертов

**Примеры запросов:**

```promql
# Количество полученных сообщений телеметрии
rate(telemetry_received_total[5m])

# Размер очереди
telemetry_queue_size

# Количество проверок зон
rate(zone_checks_total[5m])

# Ошибки автоматизации
rate(automation_loop_errors_total[5m])
```

### Прямой доступ к метрикам сервисов

#### History Logger
```bash
curl http://localhost:9300/metrics
```

#### Automation Engine
```bash
curl http://localhost:9401/metrics
```

## 🎯 Рекомендации по использованию

### Ежедневный мониторинг

1. **Утром:** Проверьте System Overview на наличие критических алертов
2. **В течение дня:** Периодически проверяйте History Logger и Automation Engine dashboards
3. **При проблемах:** Используйте детальные dashboards для диагностики

### Что проверять регулярно

✅ **Статус сервисов** - все должны быть UP  
✅ **Размер очереди** - не должен превышать 10000  
✅ **Ошибки** - должно быть минимальное количество  
✅ **Скорость обработки** - должна быть стабильной  
✅ **Алерты** - проверяйте активные алерты

### Признаки проблем

🔴 **Критические:**
- Сервис DOWN
- Очередь переполнена (>10000)
- Потеря сообщений
- Высокий уровень ошибок (>10/мин)

🟡 **Предупреждения:**
- Медленная обработка (P99 > 30 сек)
- Задержки в очереди (>300 сек)
- Увеличение количества ошибок

## 🆘 Решение проблем

### Dashboard не загружается

1. Проверьте, что Grafana запущен: `docker ps | grep grafana`
2. Проверьте подключение к Prometheus: Status → Data Sources
3. Обновите страницу (Ctrl+F5)

### Нет данных на графиках

1. Проверьте временной диапазон (может быть выбран период без данных)
2. Проверьте, что сервисы работают и экспортируют метрики
3. Проверьте Prometheus targets: `http://localhost:9090/targets`

### Метрики не обновляются

1. Проверьте интервал обновления dashboard (должен быть 15s или 30s)
2. Проверьте scrape_interval в Prometheus (должен быть 15s для dev, 30s для prod)
3. Перезапустите Prometheus при необходимости
