# ANDROID_APP_SCREENS.md
# Экраны Android-приложения и пользовательские сценарии

Документ описывает ключевые экраны приложения и флоу пользователя.


Compatible-With: Protocol 2.0, Backend >=3.0, Python >=3.0, Database >=3.0, Frontend >=3.0.
Breaking-change: legacy форматы/алиасы удалены, обратная совместимость не поддерживается.

---

## 1. Онбординг и логин

### 1.1. Welcome / Onboarding

- Краткое описание системы Hydro 2.0.
- Кнопки:
 - «Войти»
 - «Создать аккаунт» (если поддерживается).

### 1.2. Login

- Поля: email/телефон + пароль (или иной метод).
- Кнопки:
 - «Войти»
 - «Забыли пароль?»

---

## 2. Список теплиц и зон

### 2.1. Greenhouses List

- Карточки теплиц:
 - название,
 - расположение,
 - количество зон,
 - общий статус (OK / WARNING / ALERT).

- Действия:
 - выбор теплицы,
 - переход к настройкам (если есть права).

### 2.2. Zones List

- Список зон для выбранной теплицы.
- Для каждой зоны:
 - имя,
 - основная культура/тип,
 - ключевые показатели (например, средний pH, EC, температура),
 - индикатор статуса.

---

## 3. Детальный экран зоны

- Текущие значения:
 - pH, EC, температура раствора,
 - температура/влажность воздуха,
 - освещённость (если есть).
- Состояние устройств:
 - насосы,
 - клапаны,
 - свет.
- Активный рецепт:
 - название,
 - целевые значения,
 - стадия роста.
- Графики (история за выбранный период).
- Действия (при наличии прав):
 - переключение режимов (авто/ручной),
 - изменение целевых значений.

---

## 4. Экран узла (Node Details)

- Информация об узле:
 - имя,
 - тип (pH/EC/климат/свет),
 - статус (online/offline/degraded),
 - связанную зону.
- Технические детали:
 - RSSI,
 - версия прошивки,
 - uptime.
- Действия:
 - инициировать калибровку (если поддерживается),
 - перезапуск узла (через backend),
 - просмотр журнала ошибок.

---

## 5. Алерты и события

### 5.1. Alerts List

- Список открытых/исторических алертов:
 - уровень (info/warn/critical),
 - тип (pH, EC, температура, авария насоса и т.п.),
 - время возникновения,
 - зона/узел.

- Фильтры:
 - по уровню,
 - по типу,
 - по зоне.

### 5.2. Alert Details

- Полное описание алерта;
- История связанных измерений;
- Варианты действий (если есть):
 - «Подтвердить» (acknowledge),
 - «Открыть зону»,
 - «Инструкция по устранению» (если предусмотрено).

---

## 6. Provisioning: настройка узла

### 6.1. Сканирование устройств

- Экран поиска узлов:
 - сканирование Wi-Fi на наличие SSID вида `HYDRO-NODE-XXXX`,
 - отображение найденных узлов.

### 6.2. Подключение к узлу

- Инструкции пользователю:
 - выбрать узел,
 - подключиться к его Wi-Fi (возможно, автоматизация через Android API).

### 6.3. Ввод параметров

- Форма:
 - SSID основной сети,
 - пароль,
 - выбор теплицы (если уже есть аккаунт и данные),
 - выбор зоны (опционально),
 - имя узла.

- Кнопка «Сохранить и применить».

После успешной конфигурации:

- сообщение об успехе;
- предложение перейти к монитрингу соответствующей зоны.

---

## 7. Настройки пользователя

- Профиль (имя, контакты).
- Настройки уведомлений:
 - какие алерты присылать push’ами,
 - quiet hours.
- Выбор единиц измерения (°C/°F, EC/µS и т.п. при необходимости).

---

## 8. Требования к ИИ-агенту

1. ИИ должен создавать/изменять экраны так, чтобы не ломать описанные флоу.
2. Любое добавление новых экранов должно сопровождаться:
 - описанием сценария,
 - обновлением навигации.
