# LibrePCB - Быстрый гайд по использованию

## Что такое LibrePCB?

LibrePCB — бесплатное, кроссплатформенное программное обеспечение для проектирования электроники (EDA). Позволяет создавать принципиальные схемы и проектировать печатные платы.

**Официальный сайт**: https://librepcb.org/

---

## Установка

### Linux
```bash
# Ubuntu/Debian
sudo apt install librepcb

# Или скачать AppImage с официального сайта
wget https://download.librepcb.org/releases/1.3.0/librepcb-1.3.0-linux-x86_64.AppImage
chmod +x librepcb-1.3.0-linux-x86_64.AppImage
./librepcb-1.3.0-linux-x86_64.AppImage
```

### Windows
Скачать установщик с https://librepcb.org/download/

### macOS
Скачать DMG файл с официального сайта или использовать Homebrew:
```bash
brew install --cask librepcb
```

## Первые шаги

### 1. Создание нового проекта

1. Запустите LibrePCB
2. **File → New Project** (или `Ctrl+N`)
3. Выберите папку для проекта
4. Введите имя проекта
5. Нажмите **Create**

### 2. Структура проекта

Проект LibrePCB состоит из:
- **Схема** (Schematic) — принципиальная схема
- **Плата** (Board) — печатная плата
- **Библиотеки** — компоненты для использования

Все файлы хранятся в человекочитаемом формате (можно версионировать в Git!)

---

## Работа со схемой (Schematic Editor)

### Добавление компонентов

1. Откройте вкладку **Schematic**
2. Нажмите кнопку **Add Component** (или `A`)
3. Выберите компонент из библиотеки
4. Разместите на схеме кликом мыши

### Соединение компонентов

1. Нажмите **Add Net** (или `N`)
2. Кликните на контакт первого компонента
3. Кликните на контакт второго компонента
4. Соединение создано автоматически

### Полезные горячие клавиши

- `Space` — поворот компонента
- `Delete` — удаление выделенного
- `Ctrl+Z` — отмена
- `Ctrl+Y` — повтор
- `Ctrl+C` — копировать
- `Ctrl+V` — вставить
- `Ctrl+D` — дублировать
- `M` — переместить
- `R` — повернуть
- `E` — редактировать свойства

## Проектирование платы (Board Editor)

### Переход к плате

1. После создания схемы откройте вкладку **Board**
2. Компоненты автоматически появятся на плате

### Размещение компонентов

1. Выделите компонент
2. Перетащите на нужное место
3. Используйте `Space` для поворота

### Разводка дорожек

1. Нажмите **Route** (или `R`)
2. Кликните на контакт компонента
3. Ведите курсор к целевому контакту
4. LibrePCB автоматически найдет оптимальный путь

### Слои платы

- **Top** — верхний слой (компоненты)
- **Bottom** — нижний слой
- **Inner Layers** — внутренние слои (для многослойных плат)
- **Silkscreen** — шелкография (текст и маркировка)
- **Solder Mask** — паяльная маска
- **Paste** — паяльная паста (для SMD)

## Управление библиотеками

### Установка библиотек

1. **Library → Library Manager**
2. Нажмите **Download** для поиска библиотек
3. Выберите нужную библиотеку
4. Нажмите **Install**

### Создание собственных компонентов

1. **Library → Library Editor**
2. Создайте новую библиотеку или откройте существующую
3. Добавьте символ (Symbol) для схемы
4. Добавьте посадочное место (Footprint) для платы
5. Создайте компонент (Component), связав символ и footprint

---

## Проверка проекта (Design Rule Check)

### Запуск проверки

1. **Project → Design Rule Check** (или `Ctrl+D`)
2. Проверьте все ошибки и предупреждения
3. Исправьте найденные проблемы

### Типичные ошибки

- **Unrouted nets** — несоединенные цепи
- **Clearance violations** — нарушение зазоров
- **Missing connections** — отсутствующие соединения

---

## Экспорт для производства

### Gerber файлы

1. **Project → Board Production Data → Gerber Files**
2. Выберите папку для экспорта
3. Нажмите **Generate**

### Drill файлы

1. **Project → Board Production Data → Excellon Drill Files**
2. Экспортируйте файлы для сверления

### Заказ платы

1. **Project → Order PCB**
2. Выберите сервис изготовления (например, LibrePCB Fab)
3. Загрузите Gerber файлы
4. Оформите заказ

## Командная строка (CLI)

LibrePCB имеет CLI инструмент `librepcb-cli` для автоматизации:

### Проверка проекта
```bash
librepcb-cli open-project --check <путь_к_проекту>
```

### Экспорт Gerber
```bash
librepcb-cli open-project --export-gerber <путь_к_проекту> <папка_экспорта>
```

### Использование в CI/CD
```bash
# Пример для GitHub Actions
librepcb-cli open-project \
  --check \
  --export-gerber ./gerber \
  --export-drill ./drill \
  ./my-project
```

## Полезные советы

### 1. Версионирование проектов

Все файлы LibrePCB — текстовые, можно использовать Git:
```bash
git init
git add .
git commit -m "Initial project"
```

### 2. Организация компонентов

- Используйте семантические имена для компонентов
- Группируйте связанные компоненты в библиотеках
- Документируйте параметры компонентов

### 3. Оптимизация разводки

- Начните с критических цепей (питание, земля)
- Используйте автопрокладку для простых соединений
- Проверяйте зазоры для вашей технологии производства

### 4. Работа с многослойными платами

- Используйте внутренние слои для питания и земли
- Размещайте сигнальные цепи на внешних слоях
- Следите за переходными отверстиями (vias)

## Типичный рабочий процесс

1. **Создание проекта** → Новый проект
2. **Схема** → Добавление компонентов и соединений
3. **Проверка схемы** → Design Rule Check
4. **Плата** → Размещение компонентов
5. **Разводка** → Создание дорожек
6. **Проверка платы** → Design Rule Check
7. **Экспорт** → Gerber и Drill файлы
8. **Заказ** → Отправка на производство

## Решение проблем

### Проект не открывается
- Проверьте версию LibrePCB (совместимость версий)
- Убедитесь, что все библиотеки установлены

### Компоненты не отображаются
- Проверьте установку библиотек
- Обновите библиотеки через Library Manager

### Ошибки при экспорте
- Убедитесь, что все цепи соединены
- Проверьте Design Rule Check перед экспортом
